# Skein MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers-extended-cc:subagent-driven-development (recommended) or superpowers-extended-cc:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build Skein — a self-healing drone-swarm mesh network that detects jamming/cyber attacks in real time with a classic ML model trained on real CIC data, and reroutes around the damage live, shown on an interactive command-center dashboard.

**Architecture:** Two local processes talking over a WebSocket. A **Python backend** (FastAPI) runs the mesh simulator (nodes/links, traffic, routing, attack injection) and the ML detector (classic model trained on CIC), and streams full swarm state to the frontend each tick. A **Next.js/TypeScript frontend** renders the swarm map, ML detector panel, event feed, and judge-operated attack controls. The mesh never invents feature numbers — each link samples a **real CIC row** (benign vs attack) so the model's live predictions are genuinely data-driven and robust on stage.

**Tech Stack:** Python 3.13, FastAPI, uvicorn, scikit-learn, xgboost, pandas, numpy, joblib, networkx · Next.js 14 (App Router), TypeScript, Tailwind, shadcn/ui, @xyflow/react (React Flow), Framer Motion · WebSocket transport.

## Shared Contracts (defined in Task 0 — every agent reads this)

### Contract 1 — Feature schema (`backend/ml/schema.py`)

The exact ordered feature columns. The ML model trains on these; the mesh samples real CIC rows with these columns.

```python
# backend/ml/schema.py
"""Shared contract between the ML detector and the mesh simulator.
Both MUST use FEATURE_COLUMNS in this exact order."""

FEATURE_COLUMNS = [
    "flow_duration",
    "total_fwd_packets",
    "total_bwd_packets",
    "flow_bytes_s",
    "flow_packets_s",
    "flow_iat_mean",
    "fwd_pkt_len_mean",
    "bwd_pkt_len_mean",
    "pkt_len_mean",
    "avg_pkt_size",
]

# Our normalized labels. CIC raw labels are mapped onto these in train.py.
BENIGN = "BENIGN"
ATTACK_TYPES = ["DoS", "PortScan", "BruteForce"]   # everything not BENIGN
ALL_LABELS = [BENIGN] + ATTACK_TYPES

# Mesh attack-action -> CIC attack category it should sample
ACTION_TO_ATTACK = {
    "jam": "DoS",         # jamming a link looks like a flood/DoS on that link
    "hack": "PortScan",   # hacking a node looks like scanning/recon from it
}
```

### Contract 2 — WebSocket messages (`frontend/lib/types.ts`)

Backend → frontend is a `state` message every tick (~1s). Frontend → backend is a `command`.

```typescript
// frontend/lib/types.ts  — SHARED CONTRACT. Backend app.py must emit exactly this shape.

export type NodeStatus = "healthy" | "attacked" | "defending";
export type LinkStatus = "healthy" | "jammed" | "rerouted" | "down";
export type ThreatLevel = "NOMINAL" | "ELEVATED" | "CRITICAL";

export interface Prediction {
  label: string;            // "BENIGN" | "DoS" | "PortScan" | "BruteForce"
  attack_type: string | null;
  confidence: number;       // 0..1
}

export interface SwarmNode {
  id: string;               // e.g. "D1"
  x: number;                // 0..1 layout coords
  y: number;
  status: NodeStatus;
}

export interface SwarmLink {
  id: string;               // "D1-D2"
  source: string;
  target: string;
  status: LinkStatus;
  active: boolean;          // currently carrying traffic
  prediction: Prediction;
}

export interface SwarmEvent {
  t: string;                // "10:42:01"
  kind: "detection" | "reroute" | "recovery" | "info";
  message: string;
}

export interface StateMessage {
  type: "state";
  tick: number;
  threat_level: ThreatLevel;
  nodes: SwarmNode[];
  links: SwarmLink[];
  events: SwarmEvent[];     // newest-first, last ~20
}

export type CommandAction = "jam" | "hack" | "reset";

export interface CommandMessage {
  type: "command";
  action: CommandAction;
  target: string | null;    // link id for "jam", node id for "hack", null for "reset"
}
```

### Task B: Mesh simulator + backend WebSocket server  *(PARALLEL — Python)*

**Goal:** Build the swarm mesh (nodes/links), a tick loop that samples real CIC rows per link, runs detection, injects attacks (jam/hack), reroutes around damage, and streams full state over a WebSocket while accepting commands.

**Files:**
- Create: `backend/mesh/graph.py`
- Create: `backend/mesh/routing.py`
- Create: `backend/mesh/simulator.py`
- Create: `backend/app.py`
- Create: `backend/mesh/test_routing.py`, `backend/mesh/test_simulator.py`

**Context for the agent:**
- Use `ml/schema.py` (`FEATURE_COLUMNS`, `ACTION_TO_ATTACK`, `ALL_LABELS`). Load `data/sample/cic_sample.csv` into pools: `benign_rows` and `attack_rows[attack_type]`.
- Topology: 7 drones `D1..D7` with fixed layout coords (0..1) and a connected mesh (each node 2–3 neighbors). Use `networkx` for routing.
- **Per tick (~1s):** for each link, if link state is `healthy` sample a benign row; if `jammed`/attacked sample an attack row of the link's attack type. Call `detector.predict(features)`. Set link.prediction. If predicted attack → mark link `jammed`, emit a `detection` event, then call routing to find an alternate path between the affected endpoints; if found, mark the alternate path links `rerouted` + emit `reroute` event; when an attack is cleared/reset, restore `healthy` + emit `recovery`.
- **Commands:** `jam <link_id>` sets that link attacked (attack_type=DoS); `hack <node_id>` sets node `attacked` and its incident links attacked (attack_type=PortScan); `reset` clears all back to healthy.
- **Threat level:** NOMINAL (0 active attacks), ELEVATED (1–2), CRITICAL (3+).
- Self-healing must be REAL: routing actually recomputes paths around `jammed`/`down` links via networkx; don't fake it.

**Acceptance Criteria:**
- [ ] `SwarmGraph` builds 7 nodes + a connected mesh; exposes nodes/links and their mutable status.
- [ ] `routing.shortest_path(graph, src, dst, avoid=set_of_dead_link_ids)` returns a node path avoiding dead links, or `None` if partitioned.
- [ ] `Simulator.tick()` returns a `StateMessage`-shaped dict (matches Contract 2 exactly: `type,tick,threat_level,nodes,links,events`).
- [ ] `Simulator.command(action, target)` mutates state correctly for jam/hack/reset.
- [ ] On `jam` of a link that has an alternate path, within ≤2 ticks the affected traffic shows `rerouted` links and an event log entry; `recovery` emitted on reset.
- [ ] `app.py` serves `GET /` (health), `WS /ws` (streams state each tick, receives `CommandMessage`), CORS allows `http://localhost:3000`.
- [ ] Runs: `uvicorn app:app --port 8000` and a `websocat`/browser client receives state JSON ~1/sec.
